## 关键技术发现（值得汇报的洞见）

### 1. 阿里推广链接体系比文档复杂得多

**官方文档说的 `taobao.tbk.dg.material.optional`（物料搜索）实际不可用** —— 即便申请了 20 个权限包也访问不到。最终发现这个 API 已经被替换成 `taobao.tbk.dg.material.recommend`（物料库推荐），但官方文档没明确标注。

**响应结构也升级了**：从扁平 → 嵌套，字段位置全变。

**意外的好消息**：升级版 API 直接返回 `publish_info.click_url`，已经是带我们 PID 的 `s.click.taobao.com` 短链 —— **完全不需要单独调"链接 API"（邀请制权限）就拿到了转链结果**。

### 2. OCP-Catalog 协议层有个隐蔽 bug

`buildSearchProjection` 默认按白名单字段过滤，**Provider 在 `attributes` 里塞的自定义旗标会被丢弃**。Day 4 写完 4 个 patch 后 hook 死活不触发，根因就在这。

**修复**：补 Patch 005 让 projection 透传 attributes。**这条经验已经写进 [06-coding-flow.md](06-coding-flow.md) 的卡点速查表**，避免下次有人踩同样的坑。

### 3. 老 num_iid 已废弃

我们 fixture 里用的形如 `700123456001` 的数字 ID **已经被阿里淘汰**，新格式像 `J5k2X3jgfRtqN0M5bRViOmWIvta-bXRdeKMaUGomXyeGYUW` 这种 base64-ish 长字符串。任何依赖老 ID 格式的代码都得重写。

### 4. PoC 0 元成本，量产化也几乎免费

PoC 阶段的成本 = 0 元（账号免费、API 免费、容器在本地）。

正式生产化追加：
- 服务器/容器（阿里云一台轻量 ~100 元/月）
- 监控/告警（用现成飞书机器人，0）
- 数据库 Postgres（PoC 内存，生产持久化）

**单笔成交净佣金 ≈ 订单 × 佣金率 × 0.9**（扣 10% 技术服务费），佣金率因品类不同 3%-50% 不等。

## FAQ

**Q：现在演示用的商品池只有 20 个，能扩吗？**
A：能。物料库 6708 给了 20 个，再调几个其他 material_id（13366/4092/31371/3786 等）合并就到 100+。我们有 0 改动支持的能力。

**Q：动态关键词搜索能做吗？**
A：当前**不能任意关键词**（被阿里的 `material.optional` 权限挡着）。但可以：① 用物料库覆盖热门品类；② 等申请到链接 API 高级权限后切换；③ 接入百川商品搜索（独立路径）。

**Q：实际能赚多少？**
A：**取决于流量 + 佣金率**。淘宝联盟佣金率 3%-50%。1 万元 GMV 平均 1500 元佣金，扣 10% 技术费 = 1350 元到账。**关键是流量来源**——目前 PoC 阶段没有真实流量。

**Q：如果阿里改 API 怎么办？**
A：我们的 mapper 已经设计成"耦合最低"的适配层，老接口废弃我们调新接口（Day 7 就是这么处理 `material.optional → material.recommend` 的）。架构层无影响。

**Q：合规吗？**
A：当前姿态 = "deeplumen 自家内部多 Agent 使用同一个 AppKey"，**不开放给任意第三方调用**，符合阿里"API 不得开放给三方"红线。详见 [01-research.md §5](01-research.md) 三道红线分析。

**Q：能接到京东/拼多多吗？**
A：架构 0 改动支持。每接一个联盟是独立的 `provider-api`（约 1 周工作量），OCP catalog 那边完全不用动。
