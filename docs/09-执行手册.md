# Day 1-7 执行手册（你自己跑一遍）

> 目标：你从零启动到看见"用 OCP 协议查到真淘宝商品、拿到带 PID 的 s.click.taobao.com 短链"。
> 预计时间：**10-15 分钟**。所有命令都可复制粘贴。

---

## ⚠️ 重要：终端必须用 **Git Bash**，不能用 PowerShell

本手册里所有命令都是 **Bash 语法**：`\` 续行、`$(...)` 子 shell、`export VAR=value` 等。
PowerShell 会把 `curl` 劫持成 `Invoke-WebRequest`，导致 `curl -s ...` 弹出 `Uri:` 提示输入；多行命令也会因为 `\` 续行符不识别而当场报错。

**在 VS Code 终端**：右上角 ▽ 下拉 → 新建终端 → **选 Git Bash**
**独立终端**：装了 Git for Windows 后，桌面右键能看到 "Git Bash Here"

如果你必须用 PowerShell，所有 `curl` 要写成 `curl.exe`，且多行命令要重写成 PS 语法——**不建议**，照搬不会工作。

## 前提检查（30 秒）

在 **Git Bash** 终端里跑以下三条，全 OK 才继续：

```bash
docker --version    # 期望: Docker version 20.x 或更高
bun --version       # 期望: 1.3.x
node --version      # 期望: 18.x 或更高（curl/python 也需要）
```

如果有缺，先装上：
- Docker Desktop: <https://www.docker.com/products/docker-desktop>
- Bun: <https://bun.sh/>

**关键设置**：本手册的命令大量用 Python 解析 JSON 输出。**Windows + Git Bash 默认会让 Python 用 GBK 编码读 stdin**，会导致中文商品标题乱码（显示成"鎶?9?滀紒"这种）。**先跑这一行强制 UTF-8**：

```bash
export PYTHONIOENCODING=utf-8
```

想一劳永逸（写进配置文件）：

```bash
echo 'export PYTHONIOENCODING=utf-8' >> ~/.bashrc && source ~/.bashrc
```

## 第 6 步 — Agent 视角查询商品 ⭐

```bash
curl -s -X POST http://localhost:4000/ocp/query \
  -H 'content-type: application/json' \
  -d '{"query_pack":"ocp.query.keyword.v1","filters":{"provider_id":"alimama_local"},"limit":10}' \
  | python -c "
import json, sys
d = json.load(sys.stdin)
print(f'共 {d[\"result_count\"]} 个商品入库:')
for i, item in enumerate(d['items'][:6], 1):
    print(f'{i}. {item[\"title\"]}')
"
```

## 第 7 步 — Resolve 拿到 affiliate URL ⭐⭐⭐

这是**最关键的一步，给老板看就看这个**。

```bash
# 7.1 先拿一个商品的 entry_id
ENTRY_ID=$(curl -s -X POST http://localhost:4000/ocp/query \
  -H 'content-type: application/json' \
  -d '{"query_pack":"ocp.query.keyword.v1","filters":{"provider_id":"alimama_local"},"limit":1}' \
  | python -c "import json,sys; print(json.load(sys.stdin)['items'][0]['entry_id'])")
echo "Got entry_id: $ENTRY_ID"

# 7.2 Resolve(模拟 Agent 选定商品后向 catalog 要购买链接)
curl -s -X POST http://localhost:4000/ocp/resolve \
  -H 'content-type: application/json' \
  -d "{\"entry_id\":\"$ENTRY_ID\",\"agent\":{\"agent_id\":\"agt_demo_001\",\"intent\":\"shopping_purchase\"}}" \
  | python -c "
import json, sys
d = json.load(sys.stdin)
print('商品: ' + d['title'])
print()
print('ActionBinding(给 Agent 用的可点击链接):')
for b in d.get('action_bindings', []):
    print(f'  [{b[\"action_id\"]}] {b[\"label\"]}')
    print(f'    {b[\"url\"]}')
    print()
"
```

**预期输出**（Mock 模式）：

```text
商品: Tmall 电竞游戏鼠标 16000DPI RGB 有线

ActionBinding(给 Agent 用的可点击链接):
  [view_product] View product
    https://detail.tmall.com/item.htm?id=700567890005

  [buy_with_coupon] 领券购买 (满 99 元减 10 元)
    https://s.click.taobao.com/mock_coupon_xyz123

  [buy_with_tmall_coupon] 天猫专属券 (天猫专属券 满 199 减 30)
    https://s.click.taobao.com/mock_mm_coupon_xyz123
```

**预期输出**（真实模式）：上面的 `s.click.taobao.com/xxx` 会是**真实带 PID 的短链**。

**故事**：
- `view_product` 是原始淘宝链接（无 PID 归因）
- `buy_with_coupon` 是带我们 PID 的 affiliate URL —— **用户点这个下单，佣金归我们**
- `buy_with_tmall_coupon` 是天猫专属券（如果商品在天猫并配了 mm_coupon）

## 故障排查 Quick Ref

| 现象 | 原因 | 解决 |
| --- | --- | --- |
| `curl: (7) Failed to connect to localhost port 4000` | catalog 没起来或挂了 | 看第一个终端日志，看错误 |
| `Connection refused 55432` | Postgres 没起来 | `docker start ocp-catalog-postgres-wsl` |
| `Insufficient isv permissions / isv.permission-api-package-limit` | 真实模式权限包不够 | 用 Mock 模式（模式 A）或申请权限包 |
| `accepted_count: 0, rejected_count: 6` | Mapper 输出与 catalog ObjectContract 不符 | 看 [02-technical-design.md §8](02-technical-design.md) 字段映射 |
| 终端突然全是 `[material-poller]` `[order-poller]` 日志 | cron 在跑，正常 | 这是 Day 6 加的自动化 |
| **中文商品标题显示成 "鎶?9?滀紒" 之类乱码** | Git Bash on Windows 的 Python 默认 GBK 编码读 stdin | `export PYTHONIOENCODING=utf-8` 后重跑当前命令（也可参考开头"前提检查"） |
| PowerShell 里 `curl -s ...` 弹出 `Uri:` 提示 | PowerShell 把 curl 别名劫持成 Invoke-WebRequest | 换 Git Bash，或写 `curl.exe -s ...` |
| WSL 里 `cd e:/...` 静默不报错但目录没变 | WSL 不识 Windows 盘符 | 改用 `/mnt/e/...` 形式 |
