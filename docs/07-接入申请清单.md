## 你需要立刻决策的 3 件事

### 决策 1：运营主体 ── 用谁的名义申请联盟账号？

| 选项 | 适用场景 | 时间 | 风险 |
| --- | --- | --- | --- |
| **A. 个人主体**（推荐 PoC 用） | 测试演示阶段 | 即时 | 长期合规叙事弱 |
| B. deeplumen 公司主体 | 正式上生产 | 多 1-2 工作日 | 合规干净，但要走公司流程 |

**建议**：**A 先用，演示通过后切 B**。账号可以平滑迁移（把 AppKey 在公司主体下重申）。

### 决策 2：链接 API 拿不到时怎么办？

淘宝联盟的"转链 API"（`taobao.tbk.privilege.get`）是**邀请制**，需要月日均点击 5 万或 月日均 alipay 10 万才能申请。PoC 阶段不可能达标。

| 选项 | 优缺点 |
| --- | --- |
| **A. 不转链**（推荐 PoC 演示用）| 用原始 `item_url`，能演示链路但**不带 PID，不归因佣金** |
| B. 接订单侠等二房道 | 200-500 元/月起，按调用付费，能转 PID 但合规叙事弱 |

**建议**：**A**。演示阶段只为证明"AI Agent 能查能买"，归因可以晚一步。

### 决策 3：要不要顺便把多个联盟（京东、拼多多）也开了？

| 选项 | 影响 |
| --- | --- |
| **A. 只接淘宝联盟**（推荐先做） | 1 个主体、1 个账号、1 个 AppKey，简单 |
| B. 同时接京东 + 拼多多 | 每家都要分别申请，时间 ×3，但战略价值大 |

**建议**：**A**。淘宝跑通了再扩，架构可以平滑接其他联盟。

## 接入流程：4 步 + 完成判据

### 步骤 1 — 注册淘宝联盟账号

**谁做**：业务/运营
**预计时间**：30 分钟（含淘宝账号已有的情况下）
**操作**：

1. 打开 https://pub.alimama.com/
2. 用淘宝账号登录（个人主体直接用淘宝账号即可；公司主体需要单独认证）
3. 同意《淘宝联盟服务协议》注册成为"淘客"
4. 进入首页后**不要去搜商品，直接走"开发者入口"**

**完成判据**：能进入 pub.alimama.com 后台，看到"推广管理" / "效果报表" 菜单。

### 步骤 2 — 媒体备案

**谁做**：业务/运营
**预计时间**：填写 5 分钟 + 审核 1-3 工作日
**准备材料**：

| 媒体类型 | 需要什么 |
| --- | --- |
| **网站**（推荐 PoC 用） | 任意一个能访问的网站 URL（甚至 GitHub Pages / Vercel 也行）；不要求 ICP 备案 |
| **App** | App 在 iOS/Android 应用市场的链接 |
| **其他**（小程序、公众号等） | 对应平台的认证页 |

**最省事方案**：临时部署一个 `https://ocp-demo.<你的域名>` 简单的着陆页（一句话介绍"AI 智能推荐导购"即可），用它来备案。

**操作路径**：pub.alimama.com → 设置 → 媒体管理 → 新增媒体 → 填表 → 提交

**完成判据**：媒体状态从"审核中"变成"已通过"。

### 步骤 3 — 申请 AppKey

**谁做**：业务/运营（媒体备案通过后立刻可做）
**预计时间**：即时
**操作**：

1. 在 pub.alimama.com 后台找"开放平台" / "AppKey 申请" 入口
2. 选择刚才备案通过的媒体
3. 填写"应用名称"（建议：`OCP-Catalog-Alimama-Provider` 或类似可识别名称）
4. 选择"应用类型"：**淘宝联盟开发者 / 淘客类应用**

**完成判据**：拿到 2 个字符串：
- **AppKey**（一串数字，约 8-10 位）
- **AppSecret**（一串字母数字混合，约 32 位）

### 步骤 4 — 创建推广位（adzone）

**谁做**：业务/运营
**预计时间**：1 分钟
**操作**：

1. pub.alimama.com → **推广管理** → **推广位管理**
2. 点 **新建推广位**
3. 选择类型：**API推广位** 或 **导购推广位**
4. 填写名称：`OCP_PoC_Default`（或类似）
5. 保存

**完成判据**：拿到 1 个 **PID 字符串**，形如 `mm_8527634000_4234567890_3456789012`。
最后那段数字（`3456789012`）就是 **adzone_id**。

## 最终交给技术的 4 样东西（一句话清单）

完成步骤 1-4 之后，请把以下信息**复制给技术**（建议用加密渠道，比如内部 vault / 加密群聊；**不要发到公开 GitHub / 飞书公开文档**）：

```text
1. APP_KEY        =  <8-10 位数字>
2. APP_SECRET     =  <32 位字符串>
3. ADZONE_ID      =  <PID 最后一段数字>
4. PID 完整字符串  =  mm_xxx_xxx_xxx (备用，便于技术核对配套)
```

可选但有帮助：
- 备案的媒体名称（写进 OCP Provider 注册声明里）
- 备案的媒体 URL
- 一个测试用真实手淘账号（用于点链接验收时下个测试单）

## 法务 FAQ（如果法务来问）

**Q：阿里要求 API 权限不得开放给三方。我们的 OCP Catalog 是开放协议，会不会违规？**
A：当前 PoC 阶段，OCP Catalog 只服务 deeplumen 内部 Agent，**不开放给外部第三方调用**。这种"同一主体下多个内部渠道"的玩法不违反协议。如果未来要做开放平台（任意第三方 Agent 接入），需要先和阿里商务正式沟通走 "三方分成 API"（taobao.tbk.sc.*）通道。

**Q：佣金归我们还是归 Agent？**
A：PoC 阶段：佣金 100% 归 deeplumen（PID 是 deeplumen 主体下的）。生产阶段：是否给 Agent 分润、按什么比例分，是独立的业务决策，技术架构已经支持任意分配方案（按 adzone_id 隔离即可）。

**Q：用户数据怎么处理？**
A：技术上 alimama-provider 不存储任何最终用户数据。Agent 调 query 时不需要用户身份；resolve 时只接受 agent_id（Agent 自己提供的伪标识）；点击链接后用户在手淘 App 内的行为完全由阿里管。

**Q：合作终止怎么办？**
A：随时可以在 pub.alimama.com 注销账号。技术侧把 `ALIMAMA_MOCK=true` 切回 mock 模式即可，不影响 OCP 整体架构。

## 业务方常问的 FAQ

**Q：3 分钟解释一下我们要做什么？**
A：我们让 AI Agent（比如自家产品里的 AI 助手）能用一种**叫 OCP 的标准协议**查淘宝商品。Agent 不需要知道淘宝怎么对接，调一个标准接口就拿到带我们 PID 的购买链接。**用户点链接买东西，我们拿佣金**。

**Q：为什么是淘宝不是别的？**
A：(1) 商品多（淘宝 + 天猫几亿 SKU）(2) 联盟 API 最成熟 (3) 国内电商 Agent 流量场景最强。京东、拼多多是下一步。

**Q：申请过程哪里最容易卡？**
A：媒体备案审核（1-3 工作日）。**网站备案最快**，公众号/小程序要资质证明。

**Q：拿到 AppKey 还要做啥才能演示？**
A：**啥也不用做**。把 AppKey 给技术，技术 4-6 小时就能演示给你看。

**Q：演示完之后怎么往下走？**
A：3 个方向二选一或都做：
- **接更多联盟**（京东 / 拼多多 / 抖音）— 横向扩
- **真接 Agent 流量**（找一个内部 Agent 接 OCP 协议）— 纵向闭环
- **争取链接 API 邀请**（积累流量 → 给阿里看数据 → 拿到转链权限）— 解锁佣金归因
